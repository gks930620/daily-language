v10 — 데일리 연계형

일본어 기초 4권 세트(①대화 입문 ②기본 문법 ③필수 표현 ④필수 한자, 6개월)의 대안 구성 버전. 유일하게 "종이책"이 아니다: 이 저장소의 자동 생성 파이프라인(매일 30분 페이지 + 6박스 Leitner SRS)에 기초 커리큘럼을 태우는 설계. 비교 기준선: 정통 구성(정적인 책 4권을 각자 진도로).

1. 콘셉트

  • 4권을 책이 아니라 4개의 콘텐츠 스트림으로 재정의한다: 매일 아침 생성되는 한 페이지가 [오늘의 대화(1권) · 오늘의 문법(2권) · 오늘의 표현(3권) · 오늘의 한자(4권) · SRS 퀴즈] 다섯 섹션으로 구성된다.
  • 복습을 사람이 계획하지 않는다. 오늘 배운 표현 4개와 한자 2자는 그날 저녁 state/ja-basic/words.json에 box 1로 등록되고, 이후 1·3·7·14·30·60일 간격으로 페이지 상단 퀴즈에 자동으로 돌아온다 — 이 저장소의 기존 SRS 엔진(scripts/lib/srs.js)을 그대로 쓴다.
  • 책의 "정적인 부분"은 커리큘럼 시드(대화 주제 시퀀스 180일치, 문법 의존성 그래프 72항목, 표현·한자 후보 풀)뿐이다. 그날의 대화문·예문은 AI가 "어제까지 배운 문법·어휘만 사용" 제약 아래 매일 생성한다.
  • 정통 구성 대비 본질적 차이: 진도·복습·중복 방지가 전부 파일(state)과 코드로 관리된다. "어디까지 했더라"가 없다 — runlog가 기억한다.

2. 누구에게 맞나 / 안 맞나

맞는 사람

  • 이 저장소의 매일 페이지(en·ja-n1·ja-n2)를 이미 쓰고 있고, 같은 리듬(아침에 열어 30분)에 기초 과정을 얹고 싶은 사람.
  • 복습 계획 세우기를 항상 실패하는 사람 — 이 버전에서 복습은 의지가 아니라 파이프라인이다.

안 맞는 사람

  • 오프라인·종이 학습을 원하는 사람. 이 버전은 GitHub Pages 없이는 성립하지 않는다.
  • 지금 당장 시작하고 싶은 사람 — 아래 "정직한 전제"에 쓰듯, 이 버전은 파이프라인 확장(ja-basic 트랙 추가)이 선행되어야 하는 미래 설계다.

정직한 전제 (이 버전만의 특수 조건)

  • 현 시스템의 확정 철학([A6])은 "기초는 정해진 커리큘럼으로 각자, 생성 콘텐츠는 기초 완료자용"이다. v10은 이 경계를 옮기는 실험판이므로, 채택하려면 PLAN.md의 결정 변경이 먼저다.
  • 필요한 시스템 확장(개념 수준): ① scripts/lib/langs.js에 ja-basic 트랙 등록(트랙=언어×난이도 구조에 부합) ② 시드 파일을 읽어 brief에 "오늘의 진도 + 사용 가능 문법·어휘 목록"을 주입하는 prepare 확장 ③ "배운 것만 쓰기" 제약의 검증(validate 확장 — 완전 검증은 어렵고, 최소한 문법 항목 화이트리스트 검사).
  • AI가 만드는 것은 여전히 content.json 하나뿐이고, SRS 계산·상태 갱신은 전부 기존 결정론 코드가 한다 — 저장소의 절대 규칙을 그대로 상속한다.

3. 4권 구성 개요

정통 구성과의 역할 차이: 4권 = 4스트림이며, 각 스트림은 "정적 시드(사람이 한 번 설계) + 매일 생성(AI) + 자동 복습(SRS)"의 3층으로 나뉜다. 스트림마다 3층의 비율이 다르다.

권스트림정적 시드매일 생성되는 것SRS 등록
1권 대화오늘의 대화주제 시퀀스 180일치(한 줄씩)대화문 전체(6~8줄, 배운 것만 사용)등록 안 함(읽기 자료)
2권 문법오늘의 문법의존성 그래프 72항목 + 설명 원문예문 3개(그날 어휘로 갱신)등록 안 함(참조 자료)
3권 표현오늘의 표현후보 풀 720개(카드 원형)용례 문장(그날 대화와 교차)매일 4개 → box 1
4권 한자오늘의 한자빈도순 360자(카드 원형)용례 단어(배운 어휘 내에서)매일 2자 → box 1

하루 페이지 구성 (docs/ja-basic/days/<날짜>.html — JS 0줄, <details>로 정답 가리기):

  1. 오늘의 복습 퀴즈 — words.json에서 due인 표현·한자 카드 (자동)
  2. 오늘의 대화 — 시드 주제 + 생성 대화
  3. 오늘의 문법 — 시드 설명 + 생성 예문
  4. 오늘의 표현 4 — 오늘 밤 SRS 등록
  5. 오늘의 한자 2 — 오늘 밤 SRS 등록

스트림별 6개월 시드 목차

1권 대화 스트림 — 주제 시퀀스 180일 (요약)

구간주제 흐름한 줄 설명
1~30일인사→소개→숫자→쇼핑문법 G01~G12만으로 성립하는 대화
31~60일주문→길→교통→시간 약속존재문·시간 표현이 풀리며 대화가 길어짐
61~100일일상 보고→취미→날씨→감상동사문 해금 — "어제 뭐 했어요" 유형
101~140일부탁→허가→전화→트러블て형 계열 해금, 실전 밀도 상승
141~180일감정→의견→긴 대화 종합배운 전부를 섞은 종합 회화, 반말 맛보기

2권 문법 스트림 — 의존성 그래프 72항목 (주 3항목, 발췌)

G02〜ですかG01
G03〜じゃないですG01

G05〜の(소유)G01

  • 그래프인 이유: 그날 대화 생성의 제약 조건이 "오늘까지 해금된 노드 집합"으로 기계적으로 정의되기 때문. 사람이 "이거 아직 안 배웠는데"를 감시할 필요가 없다.

3권 표현 스트림 — 720개 (하루 4개 × 180일): 빈도순 + 그날 대화 주제와의 연관 우선(현 시스템의 [A7] "연관 클러스터가 잘 외워진다" 원칙 상속).

4권 한자 스트림 — 360자 (하루 2자 × 180일): 빈도순, 단 그날 표현·대화에 등장하는 자를 우선 배치(교차 노출).

시드 파일의 생김새 (개념 예시 — 사람이 한 번 만들고, prepare가 매일 읽어 brief에 주입):

// seeds/dialog-topics.json (1권) — 배열 인덱스 = Day
[
  { "day": 1, "topic": "첫인사", "must_use": [], "note": "고정 정형구만" },
  { "day": 2, "topic": "이름·국적 묻기", "must_use": ["G01", "G02"] },
  { "day": 3, "topic": "직업 소개", "must_use": ["G01", "G03"] }
]

// seeds/grammar-graph.json (2권) — 노드 + 의존성
{
  "G01": { "name": "AはBです", "deps": [], "week": 1 },
  "G02": { "name": "〜ですか",  "deps": ["G01"], "week": 1 },
  "G03": { "name": "〜じゃないです", "deps": ["G01"], "week": 1 }
}
  • prepare 확장분이 이 시드에서 "오늘의 진도"를 계산해 brief.json에 unlocked_grammar(해금 노드 목록)·today_topic을 넣으면, AI는 기존 방식대로 brief만 보고 content.json을 만든다 — 날짜·진도 계산을 AI에게 시키지 않는 현 원칙 유지.

4. 1단원 샘플 — "Day 1 페이지"가 생성된 모습

이하는 docs/ja-basic/days/2027-01-04.html(가상의 Day 1)에 렌더될 내용의 마크다운 사본이다. Day 1이므로 복습 퀴즈 섹션은 비어 있다(due 항목 0) — 대신 Day 2 아침에 어떻게 보이는지를 마지막에 함께 싣는다.

① 오늘의 대화 (1권 스트림, Day 1: 시드 주제 "첫인사")

제약 조건 표시: 사용 가능 문법 = (없음 — 오늘 G01 해금 전) → 고정 정형구만으로 구성됨

田中(たなか)： おはようございます。
다나카      : 안녕하세요. (아침 인사)

キム        ： おはようございます。
김          : 안녕하세요.

田中(たなか)： はじめまして。田中(たなか)です。
다나카      : 처음 뵙겠습니다. 다나카입니다.

キム        ： はじめまして。キムです。よろしくお願(ねが)いします。
김          : 처음 뵙겠습니다. 김입니다. 잘 부탁드립니다.

田中(たなか)： こちらこそ、よろしくお願(ねが)いします。
다나카      : 저야말로 잘 부탁드립니다.
  • 읽는 법: 소리 내어 3회. 「〜です」의 정체는 바로 아래 오늘의 문법에서.
  • (시스템 뒷면: 이 대화는 content.json의 conversation으로 생성되며, reading 필드에 전문 히라가나가 함께 온다 — 기존 ja 트랙과 같은 스키마.)

② 오늘의 문법 (2권 스트림, G01 「AはBです」)

시드 설명(정적 — 매일 같은 원문)

AはBです — "A는 B입니다". 일본어 문장의 뿌리. 어순이 한국어와 같아서 단어만 갈아 끼우면 된다. 조사 は는 [와]로 읽는다. 의존성: 없음(첫 항목). 이 항목이 해금되면 → G02(ですか)·G03(じゃないです)이 열린다.

생성 예문(동적 — 오늘의 대화·어휘와 교차)

私(わたし)は キムです。          저는 김입니다.
田中(たなか)さんは 日本人(にほんじん)です。   다나카 씨는 일본인입니다.
私(わたし)は 韓国人(かんこくじん)です。       저는 한국인입니다.
