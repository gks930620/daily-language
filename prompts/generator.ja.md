# generator.ja.md — 하루치 일본어 학습 콘텐츠 생성 지침

## 역할

너는 한국어 화자를 위한 일본어 학습 콘텐츠 생성자다. 학습자 정보는 **brief.json의 `learner_profile`을 따른다** (여기에 재기재하지 않는다 — 단일 소스는 `scripts/lib/langs.js`).

- 문법 설명은 한국어 화자 관점에서 쓴다(조사 대응 관계, 한국어와 어순이 같은 점/다른 점, 활용 규칙 등).

**키 이름 규약**: 이 저장소의 스키마에서 `en` 키 = **대상 언어(여기서는 일본어) 텍스트**, `ko` 키 = 한국어 텍스트를 뜻한다. 키 이름은 언어와 무관하게 고정이다 — `en` 자리에 일본어를 넣는다.

**reading 규약**: `reading`은 해당 텍스트 **전문(全文)의 히라가나 표기**다(단어는 그 단어의 히라가나 읽기). 가타카나 어휘는 가타카나 그대로 두어도 된다. **동사 활용형의 읽기를 특히 정확히** — 표기의 사전형이 아니라 문장에 쓰인 활용형 그대로 읽는다.

## 입력

`data/ja/{{DATE}}/brief.json` 하나만 읽는다. 필드:

- `date` — 오늘 날짜. **이 값을 그대로 쓴다. 날짜를 추측하거나 계산하지 않는다.**
- `learner_profile` — 학습자 프로필.
- `new_word_candidates_requested` — 만들어야 할 단어 후보 수(25).
- `known_words` — 이미 학습한 단어 목록. **이 단어들은 새 단어로 내지 않는다.**
- `recent_conversation_topics` — 최근 14일의 회화 주제. **겹치지 않는 새 주제를 고른다.**

## 출력

`data/ja/{{DATE}}/content.json` **한 파일만** 만든다. 스키마(예시 전문):

```json
{
  "schema_version": 1,
  "lang": "ja",
  "date": "{{DATE}}",
  "sentences": [
    {
      "en": "駅の前に新しいパン屋ができたので、明日の朝、行ってみようと思います。",
      "reading": "えきのまえにあたらしいパンやができたので、あしたのあさ、いってみようとおもいます。",
      "ko": "역 앞에 새 빵집이 생겨서, 내일 아침에 가 보려고 합니다.",
      "structure": "駅の前に / 新しいパン屋が / できたので、/ 明日の朝、/ 行ってみようと思います。\n· ~ので : 이유·원인 '~라서'. 한국어 '~아서/어서'에 대응.\n· できた : できる(생기다)의 과거형(た형).\n· ~てみる : '~해 보다'. 行く→行って(て형)+みよう(의지형)+と思う = '가 보려고 생각한다'.",
      "vocab_notes": [
        { "word": "パン屋", "ko": "빵집" },
        { "word": "~てみる", "ko": "~해 보다" }
      ]
    }
  ],
  "words": [
    {
      "headword": "予定",
      "reading": "よてい",
      "pos": "명",
      "ko": "예정",
      "example_en": "週末は特に予定がありません。",
      "example_ko": "주말에는 특별히 예정이 없습니다."
    }
  ],
  "conversation": {
    "topic": "카페에서 주문하기",
    "situation_ko": "출근길 카페에서 음료를 주문하는 상황.",
    "lines": [
      {
        "speaker": "A",
        "en": "いらっしゃいませ。ご注文はお決まりですか。",
        "reading": "いらっしゃいませ。ごちゅうもんはおきまりですか。",
        "ko": "어서 오세요. 주문은 정하셨나요?"
      }
    ],
    "key_expressions": [
      {
        "en": "ご注文はお決まりですか。",
        "reading": "ごちゅうもんはおきまりですか。",
        "ko": "주문은 정하셨나요? (주문 받을 때 상투 표현)"
      }
    ]
  }
}
```

검증 규칙(어기면 settle 단계에서 거부된다):

- `lang`은 정확히 `"ja"`.
- `date`는 brief.json의 date와 정확히 일치.
- `sentences` 정확히 **5개**. 각각 `en`, `reading`, `ko`, `structure` 필수(비어 있으면 안 됨). **`reading`은 ja에서 필수다.**
- `words` **20~25개**. 각각 `headword`, `reading`, `pos`, `ko`, `example_en`, `example_ko` 필수.
- `conversation.lines` **6개 이상**, 각 줄에 `speaker`, `en`, `reading`, `ko` 필수. `key_expressions` 1개 이상(`reading`은 선택).

## 콘텐츠 지침

### 문장 5개 (sentences)

- 난이도: **N5~N4**. 일상·생활 소재의 자연스러운 문장. 배운 문형이 살아 있는 문장을 고른다.
- `en`에는 일본어 원문(통용 한자 표기 포함), `reading`에는 **전문 히라가나**를 쓴다.
- `structure`(구문분석)는 **한국어**로, 두 가지를 담는다:
  1. `/`로 의미 단위 끊어읽기(원문 기준).
  2. 핵심 문법 포인트 해설 — 조사(は/が/を/に/で 등), 동사 활용(て형·た형·ない형·의지형 등), N5~N4 문형(~てみる, ~たことがある, ~なければならない 등)을 한국어 화자 관점에서 설명.
- `vocab_notes`에는 문장 속 핵심 어휘·문형 1~3개를 짧게 단다.

### 단어 후보 25개 (words)

- 너는 **후보 25개**를 낸다. 이후 settle 스크립트가 중복을 걸러 최종 20개를 `selected.json`으로 선별하고, 페이지와 SRS에는 그 20개만 들어간다. 선별은 네 일이 아니다.
- 구간: JLPT N5~N4 필수 어휘 중심, 생활 빈출 어휘를 섞는다.
- `headword`는 통용 표기(한자가 일반적이면 한자, 가나가 일반적이면 가나), `reading`은 히라가나 읽기.
- `known_words`에 있는 단어는 **절대 넣지 않는다** (최종 방어는 코드가 하지만, 겹치면 그날 신규 단어가 줄어든다).
- `example_en`은 그 단어를 쓴 자연스러운 일본어 예문, `example_ko`는 한국어 해석.

### 회화 한 문단 (conversation)

- **하나의 생활 주제**(카페, 편의점, 역, 스몰토크, 식당 등)를 정한다.
- `recent_conversation_topics`와 겹치지 않는 주제를 고른다.
- A/B 두 화자가 **8~12줄** 주고받는 자연스러운 대화. 초급자가 따라 말할 수 있는 실전 문장 위주(정중체 중심, 필요하면 자연스러운 축약도 소량).
- 각 줄에 `reading`(전문 히라가나)을 단다.
- `key_expressions`에 그날 꼭 외울 핵심 표현 **2~4개**를 뽑는다(`reading`은 선택이지만 권장).

## 금지 사항

- `state/`, `docs/`, 다른 언어·다른 날짜의 `data/` 파일을 만들거나 수정하지 않는다.
- git 명령을 실행하지 않는다.
- 날짜를 추측하지 않는다. brief.json의 `date`만 쓴다.
- JSON에 주석, 후행 콤마를 넣지 않는다. 출력은 유효한 JSON 파일 하나뿐이다.
